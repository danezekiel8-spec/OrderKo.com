# Launch Issue Log

Use this during pilot shifts. Keep entries short and factual.

| Time | Restaurant | Device / Browser | Area | Severity | Issue | Action Taken | Owner | Status |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  |  |  | Customer / Kiosk / Cashier / Kitchen / Admin | Low / Medium / High / Blocker |  |  |  | Open / Watching / Fixed |

## Severity Guide

- Low: visual polish or wording issue; ordering still works.
- Medium: confusing flow or recoverable staff/customer issue.
- High: orders delayed, staff workaround needed, or repeated customer issue.
- Blocker: orders cannot be placed, paid, prepared, or tracked.

## Launch-Day Rule

If a blocker appears during service, pause ordering in Super Admin or Admin, switch to paper orders, record the issue, fix after the rush, then place one full test order before resuming.
